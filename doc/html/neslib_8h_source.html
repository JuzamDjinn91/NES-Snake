183 //when display is enabled, vram access could only be done with this vram update system
184 //the function sets a pointer to the update buffer that contains data and addresses
185 //in a special format. It allows to write non-sequental bytes, as well as horizontal or
186 //vertical nametable sequences.
187 //buffer pointer could be changed during rendering, but it only takes effect on a new frame
188 //number of transferred bytes is limited by vblank time
189 //to disable updates, call this function with NULL pointer
190 
191 //the update data format:
192 // MSB, LSB, byte for a non-sequental write
193 // MSB|NT_UPD_HORZ, LSB, LEN, [bytes] for a horizontal sequence
194 // MSB|NT_UPD_VERT, LSB, LEN, [bytes] for a vertical sequence
195 // NT_UPD_EOF to mark end of the buffer
